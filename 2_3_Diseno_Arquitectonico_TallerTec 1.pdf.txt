1. Introducción 
Este documento describe la arquitectura de software de TallerTec a nivel de decisiones de diseño, 
patrones aplicados, flujos de datos entre capas y especificaciones técnicas de cada componente. Sirve 
como guía maestra para que el equipo de desarrollo implemente el sistema con consistencia y sin 
ambigüedades. 
La arquitectura de TallerTec sigue el patrón de tres capas (Three-Tier Architecture) adaptado a una 
arquitectura BaaS (Backend-as-a-Service) con Next.js 14 como framework full-stack y Supabase como 
plataforma de datos y autenticación. 
 
2. Vista General de la Arquitectura 
2.1 Principios Arquitectónicos 
• 
Separación de responsabilidades: Presentación, Lógica y Datos completamente desacopladas. 
• 
Seguridad en profundidad: Autenticación en Supabase Auth, autorización por roles en API 
Routes, RLS en BD. 
• 
Mobile-first: PWA como interfaz principal de estudiantes, con Service Worker para modo offline. 
• 
Zero-cost deploy: 100% en tiers gratuitos durante la fase académica y piloto. 
• 
Escalabilidad horizontal: Vercel Edge Functions + Supabase Pro cuando se requiera escalar.

4.3 Generación de PDF de Constancia 
El proceso de generación de PDF se ejecuta en una Supabase Edge Function para evitar timeouts en 
Next.js. Usa @react-pdf/renderer compilado para Deno: 
1. Admin aprueba solicitud desde dashboard. 
2. API Route /api/constancias/:id/aprobar actualiza estado a APROBADA. 
3. Edge Function gen-constancia-pdf es invocada con el ID de constancia. 
4. Edge Function consulta datos del estudiante, taller, horas y desglose de asistencias. 
5. Genera PDF con: logo TecNM, datos del estudiante, folio único, tabla de asistencias, firma 
digital del coordinador. 
6. Sube PDF a Supabase Storage bucket constancias-pdf con ruta: {estudiante_id}/{folio}.pdf. 
7. Actualiza registro de constancia con archivo_url y estado = GENERADA. 
8. Envía notificación por email al estudiante via Resend API.

6. Flujos de Datos Completos 
6.1 Flujo de Autenticación y Sesión 
9. Estudiante ingresa email (@matehuala.tecnm.mx) y contraseña en la pantalla de login. 
10. PWA llama a supabase.auth.signInWithPassword({ email, password }). 
11. Supabase Auth verifica credenciales y retorna access_token (JWT 7 días) + refresh_token. 
12. Next.js middleware intercepta la petición y verifica el JWT en cada request protegido. 
13. El rol del usuario se extrae del JWT (claims personalizados agregados por trigger 
handle_new_user). 
14. Redirección automática al dashboard correcto según el rol.
